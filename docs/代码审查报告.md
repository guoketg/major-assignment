### 🔴 问题1：输入校验缺失（影响评分项5：安全/隐私/成本控制）

**位置**: `backend/main.py:190-194`

```python
class ChatRequest(BaseModel):
    session_id: str
    message: str          # ❌ 无 min_length / max_length
    model: str = ""
    agent: str = "auto"
```

**问题**: `message` 字段没有长度限制。当前系统可以接受空字符串和 10 万字符的超长消息，导致：
- 空消息浪费 Token
- 超长消息可能耗尽 API 配额

**修改**:
```python
from pydantic import Field

class ChatRequest(BaseModel):
    session_id: str = Field(..., min_length=1, max_length=64)
    message: str = Field(..., min_length=1, max_length=10000)
    model: str = ""
    agent: str = Field(default="auto", pattern="^(auto|chat|research|innovate|experiment)$")
```

### 🔴 问题2：null 字符路径导致 500 错误（影响评分项5：安全护栏）

**位置**: `backend/main.py:485-499`

```python
@app.get("/download/reports/{filename}")
async def download_report(filename: str):
    filepath = os.path.join(REPORTS_DIR, filename)
    real_path = os.path.realpath(filepath)  # ❌ null 字符导致 ValueError
```

**问题**: 当 `filename` 包含 null 字节（`%00`）时，`os.path.realpath()` 抛出 `ValueError`，返回 HTTP 500。

**修改**: 在函数开头增加 null 字符检测和文件名校验：
```python
if "\x00" in filename or ".." in filename:
    raise HTTPException(status_code=400, detail="非法文件名")
if not filename.endswith('.docx'):
    raise HTTPException(status_code=400, detail="仅支持 .docx 文件")
```

### 🔴 问题4：日志脱敏缺失（影响评分项4/5：隐私保护/课程能力）

**位置**:
- `backend/agent/supervisor.py:143` — `last_msg` 完整内容记录在日志
- `backend/main.py:249` — session_id 在日志中
- `backend/tools/web_search_tool.py:91` — API 返回结果可能含敏感信息

**问题**: 日志中可能包含用户完整输入内容，不符合隐私保护要求。

**修改**: 添加日志脱敏工具函数：
```python
def sanitize_log(msg: str, max_len: int = 50) -> str:
    """脱敏日志输出"""
    s = msg.replace('\n', ' ').strip()
    return s[:max_len] + '...' if len(s) > max_len else s
```

并在 `supervisor.py:143` 改为：
```python
_log.info(f"[supervisor] last_msg={sanitize_log(last_msg)}")
```

### 🔴 问题5：Token/成本追踪完全缺失（影响评分项4：课程能力综合度 — 词元成本）

**位置**: `backend/agent/state.py:9-51`

**问题**: `AgentState` 中没有 `token_usage` 或 `total_cost` 字段，无法追踪每次对话的 Token 消耗和成本。

**修改**:
```python
class AgentState(TypedDict):
    # ... 现有字段 ...
    
    # === Token 成本追踪 ===
    token_usage: Dict[str, int]
    """Token 使用统计: {"prompt_tokens": 0, "completion_tokens": 0, "total_tokens": 0}"""
    
    total_cost: float
    """本次会话累计成本（元）"""
```

在 `llm.py` 中添加 Token 统计：
```python
# 在 LLM 调用后获取 usage
if hasattr(response, 'usage') and response.usage:
    token_usage = {
        "prompt_tokens": response.usage.prompt_tokens,
        "completion_tokens": response.usage.completion_tokens,
        "total_tokens": response.usage.total_tokens,
    }
```

### 🟡 问题6：无 Token 预算限制（影响评分项5：成本控制）

**位置**: `backend/main.py:226-266`

**问题**: 没有限制单次对话的 Token 消耗上限，可能产生意外高额费用。

**修改**:
```python
# 在 llm.py 中
MAX_TOKENS_PER_SESSION = 100000  # 每次对话最多 10 万 Token
MAX_TURNS_PER_SESSION = 50       # 每次对话最多 50 轮

def check_token_budget(session_id: str, current_usage: dict) -> bool:
    """检查是否超过预算"""
    total = current_usage.get("total_tokens", 0)
    return total <= MAX_TOKENS_PER_SESSION
```

### 🟡 问题9：Session 数量无限制（影响评分项5：成本控制）

**位置**: `backend/main.py:169`

```python
conversations: dict[str, list | dict] = {}  # ❌ 无上限
```

**问题**: 会话存储在内存中，无限增长可能导致内存溢出。

**修改**: 添加最大会话数限制：
```python
MAX_SESSIONS = 100
# 在创建新会话时检查
if len(conversations) >= MAX_SESSIONS:
    # 删除最早的会话
    oldest = min(conversations.keys(), 
                 key=lambda k: _get_history(k)[-1]["timestamp"] if _get_history(k) else "")
    del conversations[oldest]
```
