# 多 Agent 协作科研分析系统 — 功能清单

> 本文档列出系统所有可用功能及其状态。最后更新: 2026-06-12

---

## 一、基础对话功能

### ✅ 流式对话 (SSE)
- 支持用户消息的流式逐 token 输出
- 前端闪烁光标指示生成中
- 后端基于 Server-Sent Events (SSE) 协议

### ✅ 多模型选择
| 前端选项 | 映射模型 | 说明 |
|---------|---------|------|
| DeepSeek V4 Flash | `deepseek-chat` | 快速响应 |
| V4 Pro | `deepseek-chat` | 更强能力 |
| 思考模式 | `deepseek-reasoner` | 深度推理 |

### ✅ 会话历史管理
- 侧边栏列出所有历史会话
- 点击切换会话
- 创建新会话 / 删除会话
- **文件持久化** — 会话保存在 `conversations/` 目录，重启不丢
- Docker 中通过 volume 挂载 `./conversations:/app/conversations`

### ✅ 思考模式 (Reasoner)
- 思考内容（`reasoning_content`）可折叠展示
- 思考中显示 "🤔 思考中..."，完成后折叠为 "🧠 思考过程"

### ✅ Markdown + LaTeX 渲染
- 支持 Markdown 语法
- 支持 LaTeX 数学公式（`$...$` 行内，`$$...$$` 块级）
- 使用 KaTeX 渲染

## 二、Agent 系统

### ✅ Agent 流水线可视化
- **Agent 流水线面板** — 显示当前运行的 Agent 及其状态
- 每个 Agent 卡片含图标、标签、状态（⏳进行中 / ✅完成 / ❌错误）
- **工具调用日志** — Agent 调用的工具实时显示
- **折叠/展开** — 完成后自动折叠，可手动展开查看

### ✅ 手动 Agent 选择
- 输入框上方有 Agent 选择栏：
  - 🤖 **自动** — Supervisor 自动路由
  - 💬 **对话** — Chat Agent（普通对话）
  - 🔍 **调研** — Research Agent（论文检索）
  - 💡 **创新** — Innovator Agent（构思方案）
  - 🧪 **实验** — Experiment Agent（实验分析）
- 选中的 Agent 高亮显示，发消息时直接使用该 Agent

### ✅ 6 个 Agent 节点（后端 LangGraph）

| Agent | 角色 | 工具 | 触发条件 |
|-------|------|------|---------|
| 🤖 **Supervisor** | 智能路由 | 无 | 入口节点，LLM 意图分类 |
| 💬 **Chat Agent** | 对话助手 | 无 | 普通对话/闲聊 |
| 🔍 **Research Agent** | 文献调研 | `search_arxiv` | 需查论文/调研 |
| 💡 **Innovator Agent** | 创新构思 | 无 | 需构思方案 |
| 🧪 **Experiment Agent** | 实验分析 | 无 | 需设计实验 |
| 📋 **Synthesizer** | 综合输出 | 无 | 汇总各 Agent 结果 |

### ✅ LangGraph 状态图
- 使用 `StateGraph(AgentState)` 构建
- 条件路由（supervisor → agent → synthesizer → END）
- MemorySaver 检查点
- `astream_events` 流式执行 → SSE 事件映射

## 三、工具集成

### ✅ arXiv 论文搜索
- **后端**：`/arxiv/search` 端点，3 秒限流，XML→JSON 解析
- **前端**：独立论文搜索面板，展示标题/作者/摘要/分类/PDF链接
- **Agent 集成**：Research Agent 可调用 `search_arxiv` 工具
- **LangChain @tool 封装**：`tools/arxiv_tool.py`

### ✅ Word 文档生成
- **工具**：`tools/docx_tool.py`
- 功能：
  - `create_docx(title)` — 创建新文档
  - `add_section(filepath, heading, content)` — 添加章节
  - `add_table(filepath, headers, rows)` — 添加表格
- 输出目录：`backend/generated_reports/`

### 📝 待开发工具
- `parse_paper` — PDF 论文解析
- `create_diagram` — Draw.io 架构图
- `search_web` — 通用网络搜索

## 四、记忆系统

### ✅ 三层记忆架构

| 层级 | 实现 | 说明 |
|------|------|------|
| **短期记忆** | `AgentState.messages` | 当前会话的对话历史，LangGraph 节点间自动传递 |
| **工作记忆** | `memory/manager.py` + JSON 文件 | 论文归档、创新方案、实验日志，跨会话持久化 |
| **长期记忆** | ❌ 待实现 (ChromaDB) | 跨会话语义检索 |

### ✅ 工作记忆结构
```python
memory = {
    "papers_archive": [
        {"title": "...", "evidence_level": "confirmed|disputed|insufficient", ...}
    ],
    "innovation_candidates": [
        {"name": "...", "novelty": "high", "difficulty": "medium", "status": "proposed", ...}
    ],
    "experiment_log": [
        {"step": 1, "analysis": "...", ...}
    ],
    "research_topic": "...",
    "baseline_model": "...",
    "current_focus": "...",
}
```

### ✅ 记忆检索函数
- `get_context_for_agent(agent_name, memory)` — 为指定 Agent 提取相关记忆片段
- 支持 Supervisor、Research Agent、Innovator Agent、Experiment Agent

### ✅ 前端记忆面板
- **🧠 记忆** 标签页，显示当前会话的工作记忆
- 统计卡片：已归档论文数、创新方案数、实验记录数
- 论文列表（含证据等级标签）
- 创新方案列表（含新颖度/难度/状态）
- 实验日志列表

## 六、部署

### ✅ Docker Compose
```bash
docker compose up -d
# 前端: http://localhost:3001
# 后端: http://localhost:8001
```

### ✅ 本地开发
```bash
# 后端
source venv/Scripts/activate
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000

# 前端
cd frontend
REACT_APP_API_URL=http://localhost:8000 npm start
# 前端: http://localhost:3000
```

### ✅ Docker Volumes
- `./conversations:/app/conversations` — 对话历史持久化
- `./agent_memory:/app/agent_memory` — 工作记忆持久化
