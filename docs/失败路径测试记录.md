## 详细测试记录

### 1. 不存在的会话 — ✅ 通过

**场景**: 请求一个从未创建过的会话历史

**请求**: `GET /history/nonexistent_session_12345`

**响应**: 返回空历史而非报错

```json
{"session_id": "nonexistent_session_12345", "history": [], "meta": {}}
```

**结论**: 优雅降级，不会引发服务端异常

---

### 2. 路径遍历攻击 — ✅ 通过

**场景**: 尝试通过 `../` 跳出报告下载目录

**请求**: `GET /download/reports/../../../etc/passwd`

**响应**: HTTP 404 Not Found

**后端保护逻辑**: 代码 `backend/main.py:490-493` 中使用 `os.path.realpath()` 检查：

```python
real_path = os.path.realpath(filepath)
if not real_path.startswith(os.path.realpath(REPORTS_DIR)):
    raise HTTPException(status_code=403, detail="非法路径")
```

**结论**: 路径检查机制有效阻止目录遍历

---

### 3. 不存在的 API 端点 — ✅ 通过

**场景**: 请求未定义的路由

**请求**: `GET /admin/delete-all`

**响应**: HTTP 404 Not Found

**后端日志**:
```
INFO: 172.21.0.1:60218 - "GET /admin/delete-all HTTP/1.1" 404 Not Found
```

**结论**: FastAPI 默认拒绝未注册的端点

---

### 4. Agent 意图识别回退 — ✅ 通过

**场景**: 用户输入无意义的字符串 `"aaa bbb ccc"`

**后端日志**:
```
[supervisor] current=supervisor
[chat_router] tasks=0, pending=0
[synthesizer] 消息清理: 4 → 3
[DONE] cleaned history: count=4 (raw was 3)
[SAVE] session=test_intent_fallback, history_len=4, last_role=assistant, last_content_len=142
```

**流程**: Supervisor 将无意义输入判断为 "chat" 意图 → Chat Agent 正常回复 → Synthesizer 综合输出

**结论**: 系统在意图不明确时不会崩溃，安全回退到 Chat Agent

---

### 5. 空消息处理 — ⚠️ 当前行为

**场景**: 发送空字符串消息

**请求**: `POST /chat` with `{"message": ""}`

**响应**: HTTP 200，但回复内容为空

**改进建议**: 在 `ChatRequest` 模型或 `chat` 端点增加 `min_length=1` 校验

---

### 6. 超大输入处理 — ⚠️ 当前行为

**场景**: 发送 10 万字符的消息

**响应**: 能够正常处理但消耗大量 Token

**改进建议**: 在输入层添加最大长度限制（如 10000 字符）

---

### 7. 非法路径（null字符）— ❌ 需修复

**场景**: 请求包含 null 字符的路径

**请求**: `GET /download/reports/%00null`

**响应**: HTTP 500 Internal Server Error

**后端错误日志**:
```
ValueError: lstat: embedded null character in path
  File "/app/backend/main.py", line 490, in download_report
    real_path = os.path.realpath(filepath)
```

**修复建议**: 在 `download_report` 函数开头增加 null 字符检测：

```python
if "\x00" in filename:
    raise HTTPException(status_code=400, detail="非法文件名")
```

## 安全拒绝机制总结

当前系统已实现的防护措施：

| 防护措施 | 状态 | 位置 |
|---------|:----:|------|
| 路径安全检查 | ✅ | `backend/main.py` `download_report()` |
| 密钥隔离（.env + .gitignore） | ✅ | 项目根目录 |
| 模型配置模板（.env.example） | ✅ | 项目根目录 |
| API 限流（arXiv 3 秒间隔） | ✅ | `backend/tools/arxiv_tool.py` |
| LLM 调用超时（180s） | ✅ | `backend/agent/llm.py` |
| Agent 执行超时（600s） | ✅ | `backend/main.py` `chat_stream()` |
| FastAPI 输入校验（Pydantic） | ✅ | `backend/main.py` `ChatRequest` |
| 不存在端点自动 404 | ✅ | FastAPI 默认行为 |
| 会话不存在优雅降级 | ✅ | `backend/main.py` `get_history()` |

待改进的防护措施：

| 防护措施 | 优先级 | 建议 |
|---------|:------:|------|
| 输入长度限制 | 高 | 限制用户消息最大长度 |
| Null 字符路径过滤 | 高 | 在文件下载前检测 null 字符 |
| 日志脱敏 | 中 | 确保日志不打印完整 API Key |
| Token 预算限制 | 中 | 添加每次对话最大 Token 数 |
| 空消息校验 | 低 | 在 Pydantic 模型添加 `min_length=1` |
