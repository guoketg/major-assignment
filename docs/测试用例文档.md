# 自动化测试用例文档

> 本系统提供 20+ 个自动化测试脚本，覆盖 API 层、Agent 流水线、前端 UI、工具调用等核心功能。

---

## 测试运行要求

所有测试需要在以下环境中运行之一：

1. **Docker 容器运行中**（推荐）：`docker compose up -d` 后，后端在 `http://localhost:8001`
2. **本地后端运行中**：`python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000`

测试脚本均在 `tools/` 目录下。运行前需要激活虚拟环境：

```bash
source venv/Scripts/activate   # Windows Git Bash
# 或 venv\Scripts\activate     # Windows cmd
```

## 核心测试用例（≥5个）

### 1. 全面功能测试 — `tools/test_all_features.py`

**类型**: Python API 测试（requests）
**覆盖**: 8 项子测试

| 子测试 | 说明 |
|--------|------|
| 后端健康检查 | `GET /health` 确认服务正常 |
| arXiv 论文搜索 | `GET /arxiv/search` 搜索并解析 XML 结果 |
| 模型选择切换 | 3 种模型（deepseek-v4-flash / v4-pro / 思考模式）的流式响应 |
| 流式增量输出 | 验证 SSE 流式逐块返回 |
| 会话生命周期 | 创建会话 → 发送消息 → 查看历史 → 删除会话 |
| Agent 流水线 | 验证 Supervisor → Research Agent → Synthesizer 流程及工具调用 |
| Word 文档工具 | 创建文档 → 添加章节 → 添加表格 |
| 记忆接口 | `GET /memory/{session_id}` 查看工作记忆 |

**运行**:
```bash
PYTHONIOENCODING=utf-8 python tools/test_all_features.py
```

## 测试框架说明

当前测试使用以下框架/库:

| 测试类型 | 框架/库 | 用途 |
|---------|---------|------|
| API 测试 | `requests` + `unittest` | 后端 HTTP 接口测试 |
| 流式测试 | `requests` + SSE 解析 | 流式对话功能验证 |
| 浏览器测试 | `playwright` | 前端 UI 端到端测试 |
| 单元测试 | `unittest` (Python 标准库) | Agent 逻辑独立测试 |

> 注: 所有测试脚本位于 `tools/` 目录下，通过 `python tools/test_xxx.py` 直接运行。
> 后续可迁移至 pytest 框架统一管理（在 `tests/` 目录下）。
